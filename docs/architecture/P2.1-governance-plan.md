# P2.1 — Repository Health Audit Governance & Plan

**Program:** Program 2 – Platform Excellence  
**Phase:** P2.1 – Repository Health Audit  
**Status:** Permanent Repository Governance Specification  
**Repository:** Esparex Monorepo  
**Prerequisite:** ✅ Program 1 Architecture Migration Completed (M1–M2.10 + Architecture Stabilization)

---

# Purpose

P2.1 establishes the **official health baseline** for the Esparex repository after the successful completion of Program 1.

This phase is **strictly analytical**.

Its purpose is to:

- Measure repository health
- Discover technical debt
- Verify architectural integrity
- Produce a prioritized refactoring backlog
- Establish measurable baselines for Program 2

---

# Guiding Principle

> **P2.1 discovers and prioritizes issues. P2.2+ fixes them.**

During this phase:

✅ Audit  
✅ Analyze  
✅ Measure  
✅ Report  

❌ No production code modifications  
❌ No refactoring  
❌ No architecture redesign  
❌ No configuration changes  

unless absolutely required to execute the audit tooling itself.

---

# Audit Scope

The audit includes every workspace within the Esparex monorepo:
- `apps/admin`
- `apps/web`
- `backend/api`
- `core`
- `packages/contracts`
- `shared`
- `tooling`
- `scripts`

---

# Standardized Status Vocabulary

All audit deliverables and reports MUST use a fixed vocabulary:

| Status Vocabulary | Meaning |
|---|---|
| `PASS` | Meets standard completely |
| `PASS WITH OBSERVATIONS` | Acceptable with documented findings |
| `WARNING` | Needs attention during subsequent Program 2 phases |
| `FAIL` | Does not meet governance requirement (blocks phase transition) |
| `NOT EVALUATED` | Explicitly outside audit scope / tool unavailable |

---

# Audit Evidence IDs Index Specification

Audit tool executions MUST be assigned traceable Evidence IDs:

```
EV-001: verify:architecture (`npx tsx tooling/architecture/verify-architecture.ts`)
EV-002: repo:* baseline validator scripts (`node scripts/git/*`)
EV-003: npm run build / time benchmark (`build.log`)
EV-004: madge circular import check (`raw/madge.txt`)
EV-005: orphan-sweep.cjs dead code check (`orphan-sweep.cjs`)
EV-006: jscpd duplicate code check (`raw/jscpd.json`)
EV-007: depcruise dependency graph cruiser (`raw/depcruise.json`)
EV-008: AST LOC Sweep Script (files > 500 LOC)
EV-009: debt:insights tech debt engine (`node scripts/analyze-tech-debt.js`)
```

# Findings Traceability Matrix Standard

Every technical debt item MUST map directly to a refactoring task:

```
Evidence (EV-xxx) ──► Finding (DEBT-xxx) ──► Refactoring Task (REF-xxx) ──► Future PR (P2.x)
```

# Exit Criteria & Governance Decision Standard

P2.1 is complete when:

- Every workspace & bounded context has been audited.
- `audit_manifest.json` and `audit_summary.md` generated.
- All 7 Markdown reports generated with Report Versioning (1.0.0) and Evidence Metadata.
- Raw tool outputs archived under `raw/`.
- Findings Traceability Matrix fully populated.
- 0 production code modified.
- Governance Decision recorded as `APPROVED (PASS WITH OBSERVATIONS)`.
