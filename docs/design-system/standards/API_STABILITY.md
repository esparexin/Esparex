## Purpose

This document defines what guarantees the `@mad/ui` package makes to consumers at each component lifecycle state. It prevents ambiguity about breaking changes and gives downstream teams a reliable upgrade path.

---

## Lifecycle States

| State | Description |

## What Counts as a Breaking Change?

A breaking change is any modification that requires consumers to update their code to maintain the same behavior:

- Removing a prop
- Changing a prop's type
- Changing a prop's default value in a way that alters visible behavior
- Removing a named export
- Changing the HTML element rendered by a component
- Changing event handler signatures
- Removing a CSS class that consumers may target

**Not** a breaking change:
- Adding a new optional prop with a safe default
- Adding a new named export
- Internal implementation refactors with identical external behavior
- Performance improvements
- Accessibility improvements that don't alter the DOM contract

## Promotion Criteria

### `experimental` â†’ `preview`
- Implementation is complete and testable
- At least one consumer exists (internal or external)

### `preview` â†’ `stable`
- Used in at least one production application
- All consuming apps migrated to the canonical import path (Phase 2.5)
- No breaking API changes in the prior release cycle
- `tests: true`, `docs: true`, `a11y.keyboard: true` in component manifest

### `stable` â†’ `deprecated`
- A replacement component or API exists
- Migration notes documented in `Component.md`
- Deprecation notice in `CHANGELOG.md`
- Minimum 1 minor version grace period before removal

### `deprecated` â†’ `removed`
- Removal only permitted in a **major** version bump (`v2.0.0+`)
- Announced in release notes at least one version in advance

---

## Enforcement

Component status is tracked in the component manifest (`component-manifest.json`), generated by `pnpm governance:ui-baseline`.

Phase 4 will introduce CI gates that:
- Warn on imports of `deprecated` components
- Fail the build on imports of `removed` components
- Block PRs that introduce breaking changes to `stable` components
