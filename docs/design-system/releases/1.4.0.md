# @mad/ui v1.4.0 — Release Notes

**Released:** 2026-07-06  
**Phase:** 2.5 — Adoption & Component Promotion  
**Branch:** feat/ui-phase-2.5-adoption

---

## Summary

This release officially adopts `@mad/ui` components across the monorepo applications, cleans up duplicate local files and backward-compatibility shims, and promotes all 22 components to `stable` lifecycle status.

---

## Changed

### Component Promotions
All 22 component primitives, composites, and layouts are promoted to `stable` status in the component registry:
- **Primitives**: `Button`, `IconButton`, `Input`, `Textarea`, `Label`, `Badge`, `Spinner`, `Skeleton`, `Progress`.
- **Composites**: `FormField`, `Alert`, `Modal`, `Drawer`, `Tooltip`, `Table`, `EmptyState`, `ErrorState`, `LoadingState`, `ScrollIndicator`.
- **Layouts**: `Card`, `Section`, `Stack`, `Grid`.

### `@mad/ui` Shim Deletions
Removed backward-compatibility shims inside `packages/ui/src/components/` and removed their exports from `packages/ui/src/index.tsx`. All applications now consume the components directly from the stable barrel exports.

### Duplicate Files Deleted
- Deleted duplicate `ErrorState.tsx` and `LoadingState.tsx` inside `apps/admin/src/components/states/`.
- Consolidated 13 import sites in `apps/admin` to consume the standard `ErrorState` and `LoadingState` components directly from `@mad/ui`.
- Refactored `apps/web/src/components/ui/HomeSkeletons.tsx` to compose visual shimmers using the shared `@mad/ui` `Skeleton` component rather than raw custom Tailwind styles.
