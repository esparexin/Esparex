# @mad/ui v1.2.0 — Release Notes

**Released:** 2026-07-06  
**Phase:** 2A — UI Package Infrastructure  
**Branch:** feat/ui-phase-2a-infrastructure

---

## Summary

This release establishes the infrastructure layer for the shared component library. No new component implementations are included — all existing component behavior is preserved exactly. This release is purely about architecture and tooling.

---

## New

### Package Entry Points
Stable public exports now defined in `package.json` `"exports"`:
- `@mad/ui` — all React components and hooks
- `@mad/ui/icons` — curated icon barrel (lucide-react + compat aliases)
- `@mad/ui/styles/*` — CSS modules
- `@mad/ui/themes/mad` — MAD Entertrainment brand theme
- `@mad/ui/themes/default` — Neutral fallback theme (new)
- `@mad/ui/tailwind/preset` — Shared Tailwind preset
- `@mad/ui/testing` — Test utility boundary (populated in v1.3.0)

### `lib/cn.ts`
New `cn()` utility function combining `clsx` and `tailwind-merge`. Used by all Phase 2B component implementations for safe Tailwind class composition.

```ts
import { cn } from '@mad/ui'; // exported via root barrel
```

### Icon Library (`@mad/ui/icons`)
`lucide-react` integrated as the project icon library. All icons are re-exported exclusively through `@mad/ui/icons`. Applications must never import directly from `lucide-react`.

Backward-compatible aliases included:
- `CalendarIcon` → `Calendar` from lucide-react
- `SearchIcon` → `Search` from lucide-react

### Default Theme (`@mad/ui/themes/default`)
Neutral dark palette implementing the full Theme Contract. Serves as the reference implementation and starting point for future Esparex product themes.

### Folder Hierarchy
Directory structure for Phase 2B components established:
- `src/primitives/` — atomic building blocks
- `src/composites/` — composed from primitives
- `src/layouts/` — structural containers
- `src/internal/` — private implementation details (not a public entry point)

### New Dependencies
- `clsx@^2.1.1`
- `tailwind-merge@^3.3.1`
- `lucide-react@^0.511.0`

### Documentation
- `docs/design-system/standards/API_STABILITY.md` — lifecycle state and breaking change policy
- `docs/design-system/standards/THEME_CONTRACT.md` — required token specification
- `docs/design-system/future-themes.md` — planned theme roadmap
- `docs/architecture/adr/ADR-001-ui-package-boundary.md`
- `docs/architecture/adr/ADR-002-theme-contract.md`
- `docs/architecture/adr/ADR-003-public-api-freeze.md`
- `docs/architecture/adr/ADR-004-component-lifecycle.md`

### Governance
- `scripts/governance/gather-ui-baseline.ts` extended to emit `component-manifest.json`
- Component manifest schema v1.1 with fields: `id`, `group`, `owner`, `status`, `since`, `docs`, `tests`, `a11y`

## Changed

### Backward Compatibility Shims
Files in `src/components/` converted from implementations to re-export shims:
- `Button.tsx` → `export * from '../primitives/Button'`
- `FormField.tsx` → `export * from '../composites/FormField'`
- `Modal.tsx` → `export * from '../composites/Modal'`
- `Table.tsx` → `export * from '../composites/Table'`
- `EventGridSkeleton.tsx` → shim to `primitives/Skeleton`
- `Icons.tsx` → `export * from '../icons'`
- `ScrollIndicator.tsx` → `export * from '../composites/ScrollIndicator'`

All existing imports continue to work without modification.

### Package Version
`1.1.0` → `1.2.0`

## Migration Notes

No consumer changes required. All existing imports work unchanged.

New preferred import paths (available but not yet required — migration in v1.4.0):
```ts
// Existing (still works)
import { Button } from '@mad/ui';
import { ArrowRight, CalendarIcon } from '@mad/ui';

// New icon entry point (preferred going forward)
import { ArrowRight, Calendar } from '@mad/ui/icons';
```
