# @mad/ui v1.3.0 — Release Notes

**Released:** 2026-07-06  
**Phase:** 2B — Shared Component Implementations  
**Branch:** feat/ui-phase-2b-components

---

## Summary

This release implements all 22 core design system components across primitives, composites, and layouts under `@mad/ui`. Every component is built with a standard 6-file format (types, styles, code, tests, markdown contract, and index barrel) and is fully verified under `jsdom` testing.

All components are initially registered under the `preview` lifecycle state.

---

## New Components

### Primitives
1. **`Button` (UI-PR-001)**: Primary CTA trigger with full-width support, variant states (primary, secondary, outline, ghost, danger), and loading state.
2. **`IconButton` (UI-PR-002)**: Accessible square button wrapping a single icon child. Enforces required `aria-label` attribute.
3. **`Input` (UI-PR-003)**: Text input with inline prefixes/suffixes and integrated error state alerts.
4. **`Textarea` (UI-PR-004)**: Multiline text input with character counting relative to `maxLength` limits.
5. **`Label` (UI-PR-005)**: Accessible form label with required indicator and supporting inline hint text.
6. **`Badge` (UI-PR-006)**: Semantic status badge chip supporting primary visual variants.
7. **`Spinner` (UI-PR-007)**: Infinite rotate loader with screen reader labels.
8. **`Skeleton` (UI-PR-008)**: Generic pulse container placeholder for skeleton screen composition. Includes `EventGridSkeleton` as a backward-compatible named export.
9. **`Progress` (UI-PR-009)**: Accessible progress bar with `aria-valuenow` attributes.

### Composites
1. **`FormField` (UI-CP-001)**: Combines Label, input children, and error descriptors in a clean vertical layout.
2. **`Alert` (UI-CP-002)**: Semantic notices with dismiss callback actions.
3. **`Modal` (UI-CP-003)**: Accessible dialog modal with keyboard focus trapping and swipe-down-to-close behavior.
4. **`Drawer` (UI-CP-004)**: Slide drawer panel supporting bottom sheet and side orientations.
5. **`Tooltip` (UI-CP-005)**: Lightweight hover/focus popup bubble.
6. **`Table` (UI-CP-006)**: Data tables containing standard `TableHeader`, `TableBody`, `TableRow`, `TableHead`, and `TableCell` children.
7. **`EmptyState` (UI-CP-007)**: Centralized icon, title, and action for empty lists.
8. **`ErrorState` (UI-CP-008)**: Alert panels with a retry CTA for connection error state wrappers.
9. **`LoadingState` (UI-CP-009)**: Centered loader template with polite `aria-live` busy notifications.
10. **`ScrollIndicator`**: Bounce anchor animating scroll prompt.

### Layouts
1. **`Card` (UI-LY-001)**: Panel container with glassmorphism and padding variants.
2. **`Section` (UI-LY-002)**: Top/bottom vertical spacer mapping standard page sections.
3. **`Stack` (UI-LY-003)**: Flex primitive managing row/column spacing and alignments.
4. **`Grid` (UI-LY-004)**: CSS Grid layout managing responsive columns.
