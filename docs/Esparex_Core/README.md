# Esparex Core Documentation Hub

Welcome to the **Esparex Core** documentation space. This directory serves as the authoritative single source of truth (SSOT) for architecture standards, project specifications, active roadmaps, and health status indicators.

## Directory Structure

* **`PROJECT_STATUS.md`**: Living status dashboard answering "where are we today?"
* **`PROJECT_SPECIFICATION.md`**: Detailed technical specification of package boundaries, domains, and data flows.
* **`MASTER_ROADMAP.md`**: Phased priority roadmap spanning architectural closure and product quality.
* **`01_Governance/`**: Code standards, compliance checklists, and PR gating guidelines.
* **`02_Architecture/`**: Deep dive specifications on DDD, Ports & Adapters, and caching boundaries.
* **`03_Project/`**: Specific project scopes and feature specification folders.
* **`04_Audits/`**: Historical repository health audits and scoring ledgers.
* **`05_Implementation/`**: Completed migrations and system consolidation logs.
* **`06_Roadmaps/`**: Historical roadmaps and planning targets.
* **`07_Reports/`**: Phase execution and verification reports.
* **`08_Templates/`**: Standard templates for prompts, specs, and checklists.
* **`09_Archive/`**: Deprecated documentation and specifications.
