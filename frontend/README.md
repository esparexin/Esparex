# Esparex Frontend

**Purpose**: Next.js 13+ application providing the user interface for the Esparex marketplace platform.

**Stack**: Next.js 16 (App Router) + React 18 + TypeScript + TailwindCSS + shadcn/ui

---

## 🏗️ Architecture Overview

This frontend uses **Next.js 13+ App Router** with a clear separation between:
- **Routing** (`src/app/`) - File-based routing
- **Components** (`src/components/`) - Reusable UI components
- **API Layer** (`src/api/`) - Centralized backend communication
- **Services** (`src/services/`) - Business logic and data transformation
- **State** (`src/context/`, `src/hooks/`) - State management

---

## 📁 Folder Structure & Governance

### `/src/app/` - Next.js App Router
**Purpose**: File-based routing for Next.js 13+

**Structure**:
```
app/
├── layout.tsx          # Root layout
├── page.tsx            # Home page (/)
├── error.tsx           # Error boundary
├── loading.tsx         # Loading UI
├── not-found.tsx       # 404 page
│
├── admin/              # /admin route
├── browse/             # /browse route
├── post-ad/            # /post-ad route
└── [dynamic]/          # Dynamic routes
```

**Rules**:
- ✅ **ALLOWED**: Page components, layouts, loading states, error boundaries
- ❌ **FORBIDDEN**: Business logic, API calls (use services), complex state
- 🔒 **DO NOT MOVE**: App Router structure defines URL routing
- ⚠️ **PATTERN**: Pages should be thin wrappers calling components

**Example**:
```typescript
// ✅ CORRECT - Thin page component
export default function PostAdPage() {
  return <PostAdNew navigateTo={...} />;
}

// ❌ WRONG - Business logic in page
export default function PostAdPage() {
  const [data, setData] = useState();
  useEffect(() => { /* complex logic */ }, []);
  return <div>{/* complex JSX */}</div>;
}
```

### `/src/api/` - Centralized API Layer
**Purpose**: Single source of truth for backend communication

**Structure**:
```
api/
├── endpoints.ts        # API endpoint registry
├── typedClient.ts      # Type-safe API client
├── generated.ts        # OpenAPI generated types
├── admin/              # Admin API calls
├── user/               # User API calls
├── types/              # API type definitions
└── utils/              # API utilities
```

**Rules**:
- ✅ **ALLOWED**: API calls, request/response handling
- ❌ **FORBIDDEN**: Business logic, data transformation, UI logic
- 🔒 **DO NOT BYPASS**: All backend calls MUST go through this layer
- ⚠️ **PATTERN**: Components → Services → API Layer → Backend

**Example**:
```typescript
// ✅ CORRECT - API layer handles HTTP
export const getAd = async (id: string) => {
  return apiClient.get(`/ads/${id}`);
};

// ❌ WRONG - Business logic in API layer
export const getAd = async (id: string) => {
  const ad = await apiClient.get(`/ads/${id}`);
  if (ad.status === 'pending') { /* transform data */ }
  return ad;
};
```

### `/src/services/` - Business Logic Layer
**Purpose**: Data transformation and business rules

**Files**:
- `ad.service.ts` - Ad-related logic
- `user.service.ts` - User logic
- `location.service.ts` - Location logic
- `category.service.ts` - Category logic

**Rules**:
- ✅ **ALLOWED**: Data transformation, business rules, orchestration
- ❌ **FORBIDDEN**: Direct HTTP calls (use API layer), UI logic
- 🔒 **DO NOT MERGE** with API layer (different responsibilities)
- ⚠️ **PATTERN**: Services call API layer, components call services

**Difference from API Layer**:
- **API Layer**: HTTP communication, request/response
- **Services**: Business logic, data transformation, validation

**Example**:
```typescript
// ✅ CORRECT - Service handles business logic
export const getAdWithDetails = async (id: string) => {
  const ad = await getAd(id);  // API call
  const category = await getCategory(ad.categoryId);  // API call
  
  // Business logic
  return {
    ...ad,
    categoryName: category.name,
    isExpired: new Date(ad.expiryDate) < new Date()
  };
};
```

### `/src/backend/` - Frontend Helpers
**Purpose**: **FRONTEND-ONLY** utilities for server components

**⚠️ CRITICAL RULES**:
- ✅ **ALLOWED**: Data formatting, type conversions, client-safe utilities
- ❌ **FORBIDDEN**: Secrets, environment variables, database logic
- ❌ **FORBIDDEN**: Node.js server-only APIs (fs, crypto, etc.)
- ❌ **FORBIDDEN**: Direct database connections
- 🔒 **SECURITY**: This code runs in the browser - NO SECRETS

**Why It Exists**:
- Helper functions for Next.js Server Components
- Client-safe data transformations
- **NOT** a backend server

**Example**:
```typescript
// ✅ CORRECT - Client-safe helper
export const formatAdData = (ad: RawAd): Ad => {
  return {
    ...ad,
    formattedPrice: `₹${ad.price.toLocaleString()}`
  };
};

// ❌ WRONG - Server-only code
import { readFileSync } from 'fs';  // ❌ Node.js API
const SECRET_KEY = process.env.JWT_SECRET;  // ❌ Secret
```

## 🔄 Error Handling

### App Router Error Boundaries
**Files**:
- `app/error.tsx` - Page-level error boundary
- `app/global-error.tsx` - Root error boundary
- `app/not-found.tsx` - 404 handler

**Rules**:
- ✅ **USE** App Router error boundaries for page errors
- ✅ **USE** `ErrorBoundary` component for component-level errors
- ❌ **DO NOT** mix error handling patterns

**Pattern**:
```typescript
// ✅ Page-level error (app/error.tsx)
'use client';
export default function Error({ error, reset }) {
  return <div>Error: {error.message}</div>;
}

// ✅ Component-level error
<ErrorBoundary fallback={<ErrorUI />}>
  <MyComponent />
</ErrorBoundary>
```

---

### Centralized Error Handler
**File**: `src/utils/errorHandler.ts`

**Purpose**: Consistent error handling across the app

**Rules**:
- ✅ **USE** `handleError()` for all API errors
- ❌ **DO NOT** create custom error handlers
- 🔒 **PATTERN**: Centralized error classification

**Rules**:

## ⚠️ Common Mistakes to Avoid

1. ❌ **Adding pages to `src/pages/`**
   - ✅ Use `src/app/` instead

2. ❌ **Direct API calls in components**
   - ✅ Use services → API layer

3. ❌ **Business logic in `src/backend/`**
   - ✅ Move to `src/services/`

4. ❌ **Secrets in `NEXT_PUBLIC_*` variables**
   - ✅ Use server-side env vars

5. ❌ **Modifying `components/ui/` manually**
   - ✅ Regenerate with shadcn/ui CLI

6. ❌ **Importing from lucide-react directly**
   - ✅ Use icon registry
